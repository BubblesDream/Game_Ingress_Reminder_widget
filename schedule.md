# 專案開發時程表 (預估)

此時程表假設每天投入 1-2 小時開發時間。

## 第一週：基礎建設與原型
- **Day 1 (今日)**: 環境建置、Hello World 小工具跑通、熟悉工作流。
- **Day 2**: 確定 Scriptable 小工具需求 (例如：顯示資訊、資料來源 API)。
- **Day 3**: 實作基礎 UI 佈局 (使用 Stack, Text, Image)。
- **Day 4**: 串接測試 API (使用 Request 物件)。
- **Day 5**: 完成第一個原型 (MVP) 並在手機上測試。

## 第二週：功能完善與優化
- **Day 6**: 優化 UI 設計 (顏色、字體、間距)。
- **Day 7**: 加入快取機制 (Cache) 避免每次重新載入。
- **Day 8**: 處理錯誤狀態 (無網路、API 失敗)。
- **Day 9**: 加入設定參數 (args.widgetParameter) 讓使用者自訂。
- **Day 10**: 程式碼重構與模組化。

## 第三週：發布與維護
- **Day 11**: 最終整合測試。
- **Day 12**: 撰寫發布說明與分享。
- **Day 13+**: 根據使用回饋進行維護。
